# 私密自动保存的写作练习

**状态：** Accepted
**日期：** 2026-07-18

## 背景

TARA Writing Task 与 LNAT Section B 都要求学生从三个题目中选择一个，在限时环境中完成英文论证。它们不能套用客观题的选项、答案键和自动得分模型；正文又属于学生学习数据，不能为了“分析方便”被逐次按键写入事件流或默认发送给 AI。

## 决策

写作题继续复用考试无关的 Practice Session、计时、暂停、提交和私密 Learner Space，但采用显式 `responseMode: essay` 与稳定的 JSON 响应：`{ promptId, text }`。每套写作产品必须声明三道版本化原创题、官方范围锚点、时长、最大字数和可选建议字数。

编辑器在学生选择题目后启用，实时显示字数，并以 900ms debounce 保存完整 session snapshot。按键过程不产生 `answer_selected` / `answer_changed` 事件，避免把整篇正文反复复制到追加式事件日志；提交只追加一次事实型 session 事件，正文继续保存在当前学生隔离的 session snapshot。前端限制 750 词与 20,000 字符，后端仍使用既有私密行级权限和负载上限。

基础结果只返回所选题目、完整正文、字数、活跃页内用时和学习事件数量。没有经过明确 rubric、独立教师标定、版本记录与学生授权前，不生成自动分数、录取判断或写作质量标签。人工批改和可选 AI 解读作为独立的授权服务，不影响免费写作、提交和正文取回。

## 结果

- TARA 与 LNAT 共用一个可验证、响应式的写作产品，不复制两套页面。
- 学生可以恢复草稿，但事件表不会因每次按键快速膨胀。
- 写作正文继承 Learner Space 的租户隔离、导出和删除边界。
- 未来人工或 AI 反馈必须引用明确的提交版本和授权，不能从浏览器草稿旁路读取。
- 写作内容仍需独立教师与实机终审；编辑器上线不等于评分产品上线。
