# 2026-07-17: 真题最终采用原生逐题交付

**背景：** 直接嵌入完整 PDF 虽能避免自动转写公式出错，但移动端体验差，也会把整套资料暴露为可直接访问的静态文件。数学与科学题还需要可靠表达公式和图形。

**选项：**

1. 继续嵌入 PDF：实现简单、原貌准确，但体验和内容保护都不满足目标。
2. 每题使用整页截图：减少完整 PDF 暴露，但可访问性、移动端清晰度和知识标注较差。
3. 语义文本 + KaTeX + 审核 SVG：维护成本较高，但最适合在线作答、响应式显示、知识图谱和后续解析。

**决策：** 选择第三种作为最终形态。MinerU、Poppler 或其他解析器只生成不可发布草稿；每套题必须经过题面、选项、图形、答案和来源页核验。结构化试卷禁止 PDF block，公开 manifest 不提供完整 PDF 路径。

**实施进度：** 已审计页码表直接驱动 extraction；旧卷的一页多题使用显式题号—来源页映射。18 套、360 道题均已转为 typed structured content，公式使用 KaTeX，图形使用仓库内安全 SVG；manifest 全部为 `structured`，公开完整原卷 PDF 已清除。staging bundle 继续保留为不可发布的审计输入。

**影响：** 生产环境中的原始文件进入私有对象存储；客户端按 entitlement 获取最小题目资源，并加入短时访问、限速、异常抓取检测、水印和审计。此方案只能降低批量搬运风险，不能阻止用户截取浏览器已经显示的内容。
