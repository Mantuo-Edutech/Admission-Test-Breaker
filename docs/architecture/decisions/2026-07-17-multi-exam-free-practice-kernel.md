# 2026-07-17: 共享免费练习内核与考试适配器

**背景：** TMUA 已有完整在线真题，但 ESAT、LNAT 和 UCAT 的模块、题量、时长、响应形式和评分规则差异很大。复制四套练习系统会造成行为、数据和权限边界漂移；把所有差异塞进一个组件又会快速失控。产品同时要求真题免费、基础结果免费，并把完整解析和个性化解读作为可收费的增值能力。

**选项：**

1. 每个考试复制一套练习页面和会话模型——上线初期直接，但计时、事件、隐私和修复会分叉。
2. 所有考试使用一个包含大量条件分支的通用页面——文件数量少，但特殊题型难以测试和维护。
3. 共享领域内核 + 版本化考试结构 + 专属响应/渲染适配器——复用会话与事件，同时让考试差异显式、可测试。

**决策：** 采用选项 3。Practice Session、计时、标记、提交和学习事件保持考试无关；`assessment-structures.json` 保存各考试模块规则；响应适配器负责单选、多选和写作的稳定序列化，专属 UI 与评分在相应内容发布时独立验收。题目、答案和事实型结果是 public；逐题完整解析、错误类型与个性化解读是独立 entitled 资源，由服务端权限控制。

**影响：**

- TMUA 继续使用现有 18 套原生题卷，但存储键和页面数据不再写死 20 题/75 分钟。
- ESAT、LNAT、UCAT 可以先交付准确结构和内部目录，但没有合规题卷时不能显示虚假开始入口。
- LNAT 写作不得套用客观题评分；UCAT 部分得分必须有单独验收规则。
- 深度解析不得打进公开前端 bundle 后只做视觉隐藏；entitlement 与资源发布状态必须同时成立。
- 新增考试时必须更新结构 registry、适配器测试、官方来源登记和机器可读 feature manifest。
