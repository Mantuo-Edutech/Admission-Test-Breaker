# ADR: 逐题解析必须绑定经视觉核验的题面版本

- 日期：2026-07-18
- 状态：Accepted
- 范围：原生在线题库、教研解析、私有内容交付

## 背景

Early Specimen Paper 1 已经具备结构化题面和官方答案映射，但制作解析时重新验算第 3 题，发现当前选项与答案标签不一致。追溯原 PDF 后确认答案 B 正确，旧转录却把原来的五个正数选项错误写成六个不同分数。继续逐页检查后，20 题中共有 12 题存在选项、表达式、次数或区间条件错误。

这说明“文件已导入、答案序列正确、结构 schema 通过”仍不足以发布教学解析。解析会放大题面错误，并让学生错误地怀疑自己的推理。

## 决策

逐题解析只能绑定一个明确的试卷 revision，并同时满足：

1. 题面和选项逐页对照原 PDF；视觉公式不能只依赖文本抽取；
2. 每道题的数学结论与官方答案标签一致；
3. 内容资产记录题目、答案和 worked-solution 三份来源 SHA-256；
4. 自动测试锁定高风险题面的关键公式、选项数量、答案序列和解析资源 ID；
5. 解析以本地 canonical JSON 为唯一编辑源，由确定性构建器生成数据库 migration；
6. 结果页只有在试卷显式声明已发布解析资源时才显示解锁承诺；通用 access policy 不得自动产生营销文案。

## 结果

`tmua-specimen-p1-worked-explanations-v1` 成为第一套符合该门槛的逐题解析产品。基础题目、答案与事实结果继续公开；解析正文只由 `get_entitled_content_resource` 在有效 `tmua-deep-review` 或 `tmua-full-access` 下返回。其他试卷在完成同样审核前只显示免费基础结果，不展示空的付费承诺。

代价是逐题发布速度降低，但能够把导入错误挡在教学产品之外，并给未来 PDF/MinerU/Agent 导入提供可重复的独立验证层。
