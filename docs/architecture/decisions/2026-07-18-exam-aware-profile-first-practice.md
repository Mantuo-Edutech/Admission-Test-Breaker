# 2026-07-18：按考试建立本人背景档案后再进入练习

**状态：** 已采纳；本地 Web、数据库与 HTTP 已验证，真实设备和生产云端仍待完成。

## 背景

学生在进入题目以前需要先说明课程体系、年级、已学学科、练习经历和每周时间。原先所有 `/practice/:paperId` 都复用了 TMUA 课程档案门禁，导致直接打开 UCAT、LNAT 或 TARA 题目时被错误带入 TMUA 数学课程页。这既破坏用户路径，也会让不同考试的数据含义混在一起。

ESAT 还具有专业决定模块、模块再决定课程覆盖的独特路径，不能被通用背景档案取代。

## 决策

练习路由先从版本化试卷注册表解析考试，再选择对应门禁：

- TMUA 使用现有精确课程模块档案和知识覆盖；
- ESAT 使用“专业与模块 → 课程信息”的确定性计划；
- TARA、LNAT、UCAT 各自保存一份最小背景档案，包括入学年份、课程体系、当前年级、学科、练习经历和每周时间；
- 保存 TARA、LNAT 或 UCAT 档案后先进入独立的 `/exams/<exam>/preparation` 起点定位页，再由学生选择整库或具体模块练习；
- 起点定位使用版本化固定规则逐模块返回可迁移课程依据、需要补充或确认的中英文主题、第一轮建议时间和对应短诊断，不调用实时 AI；
- 未识别试卷不伪造档案要求，由内容注册表验证负责拦截。

档案阶段不调用实时 AI，也不收集姓名、电话或微信。未登录时按 `GuestSpace + examId` 存在当前设备；登录后只能通过受认证 RPC 归入本人 `LearnerSpace + examId`。一个考试的档案不能解锁另一个考试。

## 数据与权限边界

`assessment_background_profiles` 以 `(learner_space_id, exam_id)` 为主键，启用并强制 RLS。浏览器只有本人读取权限，没有直接写权限；保存和删除通过验证 RPC 完成，复用不可被第二个账号认领的 Guest claim。本人 JSON 导出包含所有考试背景档案，账号删除通过 Learner Space 外键级联清除。

该档案只描述学生提供的背景，不是能力诊断、考试分数、录取概率或训练时长 Benchmark。起点定位中的时间只覆盖题型熟悉、基础复习和一轮短诊断，不代表总训练时长或个人能力预测。老师、家长、冰冰和 Agent 不因邀请码或内容 entitlement 获得读取权。
