# 2026-07-19：练习结果与增值解析通过产品目录建立闭环

## 背景

免费在线练习和基础结果已经属于 Practice；逐题解析属于服务端 entitlement 资料。此前结果页通过题目里的资源 ID 推断是否显示解析，并写死题量和销售文案。这会让页面在内容尚未发布时产生虚假承诺，也无法让其他考试复用同一条转化旅程。学生从结果页取得邀请码后还会在注册、登录或邮件确认完成时丢失原来的结果地址。

## 比较的方案

1. **每个结果页单独写解析卡片。** 实现快，但产品关系、题量、入口和文案会继续漂移，淘汰。
2. **只根据题目资源 ID 推断产品。** 能发现正文，但不能证明该正文已进入公开产品目录、已有真实权限包和学生入口，淘汰。
3. **产品目录声明练习关系，并与题目映射交叉验证。** 由同一产品事实源驱动标题、指标、权限包和入口；练习内容与 entitlement 资源必须一致，否则失败关闭。当前采用。

## 决策

- `review-notes` 产品可以用 `relatedPracticeIds` 声明其对应练习；结果页不得再写死某个考试、年份或题量。
- 只有同时满足“目录存在公开 invite 产品、所有题目映射同一解析资源、产品 ID 与资源 ID 完全一致”时，结果页才显示增值解析。缺少产品时不做销售承诺；关系冲突时显示服务错误并停止解锁。
- 免费题目、答案与基础结果不受该关系影响。权限只控制逐题解析或后续个性化解读。
- 未登录或未授权学生可选择“联系冰冰获取邀请码”或“已有邀请码”；前者使用现有二维码桥接，后者进入邀请码验证。
- 来源结果地址只允许站内绝对路径。它保存在当前浏览器 session，经过写入和读取双重校验；外部 URL、协议相对地址、反斜杠和换行均拒绝。
- 注册、登录或邮件确认完成并核销邀请码后，学生返回原结果页；兑换完成会同时清除邀请码和返回地址。没有安全返回地址时才进入通用完成页。
- 该返回地址不是授权证据。服务端仍以当前账号的 entitlement、资源发布状态、有效期和撤销状态决定是否返回正文。

## 验证

- `pnpm verify:content-products` 检查目录字段、产品状态、路由和证据。
- `pnpm verify:entitled-content` 检查产品、题目资源映射和服务端正文一致性。
- 账户单元测试覆盖安全地址、跨注册/登录/确认保存和兑换后清理。
- `tmua-deep-review-funnel` Playwright 旅程在电脑、iPad、手机验证“提交结果 → 查看真实产品 → 联系冰冰/已有邀请码 → 返回原结果”的布局和链接。
- 数据库/HTTP 验证继续覆盖 locked、available、revoked 和第二账号拒绝；前端 E2E 不替代服务端权限证据。

## 影响与后续复查

首个落地关系是 `tmua-specimen-p1` 到 `tmua-specimen-p1-worked-explanations-v1`。其他练习只有在真实解析正文发布、产品目录登记、题目映射和权限验证全部完成后才能加入。生产仍需真实冰冰账号、MFA、正式邮件、云端 entitlement 和真机 UAT，因此本决策不改变 100 人 Beta 的 not-ready 结论。
