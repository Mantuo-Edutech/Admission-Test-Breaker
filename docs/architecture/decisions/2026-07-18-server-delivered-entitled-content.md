# 2026-07-18：私有资料采用服务端结构化正文交付

## 背景

邀请码、注册和 `user_entitlements` 已经能够工作，但只有权限记录、没有真实资料，不构成产品闭环。把私有 JSON import 到 React 或把 PDF 放入 `public/`，即使按钮被隐藏，正文仍会进入公开构建产物，无法满足“只有当前学生账号有权取得”的边界。

此决策作出时，首个真实资产选择 `TMUA 六周精确训练计划`：它有六周、30 次训练、错误分类、课程体系调整和诚实 Benchmark 边界，适合验证原生网页交付，同时不把尚未完成的 360 道逐题深度解析写成现有能力。后续新增的第二项真实资产是 `Early Specimen Paper 1` 全 20 题深度解析；两者沿用同一服务端交付边界。

## 比较的方案

1. **前端 bundle + UI 隐藏。** 实现最短，但任何访问者都可以下载 JavaScript chunk 取得正文，淘汰。
2. **公开 PDF + 难发现 URL。** 无法撤销、无法审计，URL 泄露后权限失效，淘汰。
3. **私有对象存储 + 短时签名 URL。** 适合大型 PDF、图像和压缩包；当前纯结构化文本会增加对象同步和渲染版本管理，留作后续文件资产方案。
4. **受保护数据库 payload + entitlement RPC。** 正文不进入前端 bundle，服务端根据 `auth.uid()`、资源发布状态、package 绑定、有效期和撤销状态决定是否返回；适合原生网页结构。当前采用。

## 决策

- 版本化源文件保存在 `content/notes/tmua/six-week-review-plan-v1.json`，运行时 schema 拒绝残缺正文。
- 部署 migration 将经过复核的 revision 写入 `content_resource_payloads`；浏览器角色没有该表的 `select` 权限。
- `get_entitled_content_resource(resource_id)` 是唯一学生读取路径。未登录不能执行；已登录但无权限或已撤销返回空结果，不泄露正文。
- 前端明确区分 `unauthenticated`、`locked`、`available` 与服务错误；网络错误不能伪装成“未解锁”。
- `tmua-full-access` 只控制资料读取，不产生冰冰、老师、家长或 Agent 对 Learner Space 的任何 Grant。
- 可信运营服务角色只有读取 entitlement 与更新 `revoked_at` 的最小权限，不获得学生浏览器角色的写入口。

## 验证

- canonical source 的 SHA-256 固定为 `9c1430c1fa10ebe313483b367a65f0516381924528a76638107c2f48298fc438`，测试同时核对源文件、migration 和服务响应。
- pgTAP 证明未授权、授权和撤销状态，且浏览器不能直接查询 payload 表。
- 真实本地 HTTP 流程证明邀请码核销后得到完整 6 周/30 次正文，第二账号和撤销后的原账号均得到空结果。
- 页面测试证明锁定状态不渲染正文，授权状态才显示所有周次。

## 影响与后续复查

这个选择适用于结构化网页资料，不等于绝对 DRM；已经显示在浏览器中的文字仍可被截图或手工复制。大文件应迁移到私有对象存储与短时授权下载，并增加访问审计、速率限制和异常抓取检测。生产项目绑定后还必须复验跨设备登录、权限撤销传播和移动端长文 UAT。
