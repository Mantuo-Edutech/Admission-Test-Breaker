# ADR：由学生签发的老师／家长协作 Grant

**日期：** 2026-07-19
**状态：** 本地实现并验证；生产部署与真实双账号 UAT 待完成

## 决策

老师和家长不会因为身份、满托资料邀请码、冰冰运营权限或内容 entitlement 自动获得学习数据。学生在 `/account/sharing` 选择协作者类型、考试、有效期与五项独立权限后，系统生成只显示一次、七天有效的协作码：

- `progress:read`：只读汇总进度；
- `responses:read`：读取具体作答与逐题用时；
- `annotations:write`：添加批注；
- `plans:write`：制定训练计划；
- `assignments:write`：布置练习。

接收者必须登录自己的账号并在 `/collaboration/redeem` 兑换。数据库只保存 code digest，不保存对方邮箱；不能自行兑换自己的协作码。每个 Grant 限定考试和 1–180 天有效期，撤销后新请求立即失败。成功的敏感读取与写入进入学生可见审计，失败请求不伪装成成功。

## 数据与服务边界

四张 `private` 表保存邀请、Grant、协作内容和审计；浏览器角色没有任何原始表权限，只能调用受认证的 security-definer RPC。汇总进度与具体作答使用两个 RPC，避免低敏权限隐式升级为答案读取。批注、计划和作业分别检查对应写权限。学生数据导出 schema v4 包含邀请、Grant、协作内容和审计，但不包含 code digest。

前端分为学生管理、协作码兑换、已授权学习空间列表和单个协作工作区四个路由。工作区先从服务端重新取得当前有效 Grant；仅隐藏按钮不构成安全边界。

## 选择一次性协作码的原因

首版不要求学生填写老师或家长邮箱，避免额外收集联系方式、账号枚举和错误邀请。协作码由学生通过可信渠道转交，并在接收者账号上一次性绑定。未来可以增加受验证的直接邀请，但不得改变学生签发、逐项授权、限时、可撤销和可审计的底层模型。

## 结果与未完成项

本地 pgTAP 已证明老师与家长正向权限、跨租户拒绝、scope 不继承、考试隔离、一次性兑换、数据导出、审计和即时撤销。生产前仍需部署 migration，为双方账号启用 MFA，以真实手机、iPad、电脑和屏幕阅读器完成双账号 UAT，并演练协作者页面保持打开时学生撤销授权。

外部 Agent 尚未接入。将来飞书、Hermes、OpenClaw 或其他 Agent 只能映射到同一 Grant 模型，不能得到数据库查询能力或从内容 entitlement 推导学习数据权限。
