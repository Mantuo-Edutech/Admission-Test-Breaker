# Admission Test Breaker 产品宪章

**状态：** 生效
**日期：** 2026-07-13
**发起者：** 满托
**适用范围：** 产品、数据、内容、商业化、集成和开发验证

## 1. 使命

让英国大学入学考试不再因为信息不对称、资源集中和缺乏客观反馈而令人焦虑。

Admission Test Breaker 不是一个只展示题目的题库。它是学生拥有的学习基础设施：任何学生都能迅速开始高质量训练，持续积累属于自己的学习证据，在明确授权下与老师、家长或 Agent 协作，并获得诚实、可解释的时间与能力判断。

## 2. 核心承诺

### 2.1 对学生

- 在数分钟内开始真实、有质量的练习，不以购买课程为前置条件。
- 每次作答、修改、停留、回看、错误和训练频率都能形成可解释的学习证据。
- 学生拥有自己的 `Learner Space`，并决定谁能在什么时间、以什么权限访问。
- 系统不通过虚假倒计时、伪造排名或不透明模型制造焦虑。
- 系统给出的结论必须能追溯到数据、算法或 AI 运行版本。

### 2.2 对老师与家长

- 只有获得学生授权后才能进入协作关系。
- 查看、批注、制定计划、布置练习等权限分开授予，不捆绑为“全部可见”。
- 能依据同一份学习证据协作，同时看不到授权范围之外的私密信息。

### 2.3 对贡献者

- 题目、元数据、审核记录和工具有清晰的贡献、校验与版本流程。
- 内容可以被社区改进，但发布必须经过来源、结构、答案与质量验证。
- 内容的开放许可与学生数据的私密性是两条不同边界，不能混淆。

### 2.4 对满托

- 满托以发起者、维护者和可信服务提供方的身份公开出现。
- 免费工具以真实价值建立信任和自然流量，不使用暗门、虚假中立或数据绑架获客。
- 当学生需要深度解释、人工教研、课程或顾问服务时，可以主动选择满托。
- 商业价值来自更好的解释、服务和执行，不来自限制基础训练或出售学生数据。

## 3. 产品参与者

| 参与者 | 核心目标 | 默认数据权利 |
| --- | --- | --- |
| 学生 | 练习、理解、计划、获得帮助 | 拥有并控制自己的学习空间与授权 |
| 老师 | 批注、计划、布置练习、跟踪被授权进展 | 默认无权；按学生授权获得具体权限 |
| 家长 | 理解进度与投入时间、支持学生 | 默认无权；按学生授权获得具体权限 |
| 内容贡献者/审核者 | 增加、修订和验证高质量内容 | 仅访问内容工作区，不访问学生数据 |
| 外部 Agent/平台 | 在授权下读取或执行特定任务 | 限时、限域、可撤销、可审计 |
| 满托运营者 | 维护平台、提供可选服务 | 运维最小权限；业务访问必须有合法目的与审计 |

学生可以同时授予老师和家长不同权限。角色只表达关系，真正的访问能力由具体授权决定。

## 4. 产品飞轮

```text
开放可信内容
→ 低门槛真实训练
→ 结构化学习证据
→ 公平 Benchmark 与个性化解读
→ 老师/家长/Agent 经授权协作
→ 更好的训练结果与内容贡献
→ 对平台和满托的长期信任
```

这个飞轮的抓手不是“更多页面”，而是三类可复用资产：高质量内容、学生长期事件数据、基于授权产生的可信协作关系。

## 5. 产品与商业分层

### 公益基础层

- 练习题访问与基础训练；
- 基础计时、错题、历史和个人趋势；
- 数据导出、删除和授权管理；
- 社区内容贡献和质量记录；
- 诚实标示样本条件的基础 Benchmark。

### 可收费解释层

- AI 深度复盘、长期能力画像与训练建议；
- 多次考试或多类能力的综合解释；
- 老师工作台中的高级编排和批量反馈；
- 满托人工教研、课程、顾问或其他明确标价的服务。

“训练免费、解读可收费”的前提是：免费层仍然完整可用，付费结论不伪装为唯一真相，用户能知道使用了哪些数据和模型。

## 6. 数据原则

### 6.1 数据所有权与控制

- 学生是学习数据的拥有者；平台是受托处理者和系统运营者。
- 学生可以查看、导出和删除数据，并撤销未来访问。
- 撤销不会抹除已经发生的合法审计事实，但会终止新的读取和执行能力。
- 未成年人的同意、监护和数据处理应按实际运营地区的法律单独实现，不能仅靠产品文案替代。

### 6.2 收集边界

记录对学习解释有意义的动作，例如会话开始、题目查看、答案选择或修改、标记、提交、用时和训练频率。默认不记录设备指纹、无关浏览行为、自由键入内容或跨站追踪。

### 6.3 公共和私密边界

- 可公开：题目内容、来源元数据、标签体系、验证规则、匿名聚合方法和开源代码。
- 必须私密：账户、会话、作答、时间、错误、授权、老师批注、AI 输入输出和审计记录。
- 任何聚合发布都必须满足最小样本、去标识化和重识别风险检查。

## 7. 公平 Benchmark 原则

Benchmark 的目标是帮助学生估计训练投入和理解相对位置，而不是制造确定性。

每个结果至少绑定：考试和版本、适用人群、样本量、时间窗口、数据质量、计算版本与置信信息。样本不足时展示“样本积累中”，不得用模拟百分位冒充真实人群结论。不同备考阶段、考试环境或完成条件的数据不得无解释混合。

## 8. AI 与 Token 原则

AI 的价值来自高质量、经授权、结构化的上下文，而不是单纯调用更多 Token。

- 所有 AI 能力通过可配置 AI Gateway 调用；业务模块不绑定单一模型厂商。
- 每次运行记录调用目的、输入数据范围、提示词/策略版本、模型、成本、输出和人工反馈。
- 学生授权决定 AI 可以读取什么；老师或外部 Agent 不能扩大该范围。
- 训练过程可以免费，深度解释可以收费；价格与 Token 成本分开呈现产品价值。
- 未来支持平台密钥、租户密钥和用户自带密钥，但密钥不进入前端代码或学习事件。
- AI 结论是带版本的解释，不覆盖原始事实或人工批注。

## 9. 非目标

- 通过封锁真题或基础分析强迫付费；
- 将所有教育关系压缩为“家长监控学生”；
- 未经授权向老师、满托顾问或第三方展示个人学习数据；
- 在样本不足时给出看似精确的百分位、预测分数或所需训练小时数；
- 以微服务数量、AI 功能数量或事件数量替代真实学习价值；
- 让开源仓库成为任何生产学生数据的副本。

## 10. 产品成功指标

成功指标按价值链组织，而不是只看访问量：

| 阶段 | 领先指标 | 质量约束 |
| --- | --- | --- |
| 体验价值 | 首次访问到开始第一题的时间、完整练习完成率 | 无答案泄漏、无恢复丢失 |
| 学习价值 | 周训练频率、错题复练率、个人能力趋势稳定性 | 结论可追溯、不过度推断 |
| 协作价值 | 学生主动创建的授权关系、计划完成率 | 权限最小化、撤销及时生效 |
| 内容价值 | 通过验证的题目数、修订周转时间、贡献接受率 | 来源与审核状态完整 |
| 商业价值 | 解读购买转化、满托服务主动咨询率、复购/留存 | 免费层不降级、无数据出售 |

## 11. 决策门

任何新功能在进入开发前必须回答：

1. 它为哪一位参与者减少了什么焦虑或摩擦？
2. 它读取和产生哪些数据，学生如何控制？
3. 它属于免费训练、付费解释还是满托服务？
4. 它如何被独立验证，失败时如何安全降级？
5. 它是否遵守系统模块边界，是否为未来 API/Agent 接入留下明确契约？

无法回答这些问题的功能不进入发布范围。
